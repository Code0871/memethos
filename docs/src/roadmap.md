# Roadmap MindMap v0.1.0 (in Russian)

> **Цель релиза:** Создать работающее хранилище эпизодической памяти с базовой статистикой, поддержкой момента, цепочки, одного типа векторного индекса и простого гибридного поиска

---

## Этап 0: Фундамент и архитектура (Подготовка)
- [ ] **Архитектурная схема** — создать общую диаграмму компонентов: Ядро (Moment/Chain), Хранилище (диск), Векторный индекс (HNSW), B-Tree индекс (метаданные), WAL
- [ ] **Определить и реализовать структуры данных:**
    - [ ] Структура `Moment` (поля: GUID, вектор, метаданные JSON, временная метка, ссылка на GUID Chain)
    - [ ] Структура `Chain` (поля: GUID, вектор цепочки, список GUID моментов, временные границы, агрегированные метаданные)
- [ ] **Определить API векторного пространства:** один интерфейс для in-memory хранения векторов (добавление, удаление, поиск k ближайших соседей)
- [ ] **Определить схему хранения статистики** (какие метрики собираем: количество запросов, время ответа, размер кеша, количество моментов/цепочек)
- [x] **Определить схему векторных пространств** 

---

## Этап 1: Реализация ядра (Функциональный минимум)
- [ ] **Хранилище на диске:**
    - [ ] Реализовать запись и чтение `Moment` и `Chain` в бинарный файл / через `mmap`
    - [ ] Реализовать **WAL** для каждой операции записи/удаления/обновления с возможностью восстановления состояния при старте
- [ ] **Кеш в оперативной памяти:**
    - [ ] Реализовать `in-memory` кеш для `Moment` (последние N по времени или по частоте запросов, хотя бы простой LRU)
    - [ ] Реализовать `in-memory` кеш для `Chain` (аналогично)
- [ ] **Векторный индекс:**
    - [ ] Реализовать **один тип индекса** (HNSW) для хранения векторов и поиска k ближайших соседей
    - [ ] Интегрировать его с операциями добавления/удаления `Moment` и `Chain`
- [ ] **Индекс по метаданным:**
    - [ ] Реализовать **B-Tree индекс** для быстрого поиска по метаданным (по временным меткам, тегам, числовым полям)
- [ ] **Гибридный поиск (базовый):**
    - [ ] Реализовать запрос, который: 
        1. ищет моменты и цепочки по вектору через HNSW,
        2. фильтрует моменты и цепочки по метаданным через B-Tree, 
        3. возвращает топ-K результатов
- [ ] **Сбор статистики (минимальный набор):**
    - [ ] Количество моментов и цепочек в хранилище
    - [ ] Размер кеша в байтах (занимаемая память)
    - [ ] Среднее время выполнения поискового запроса (в миллисекундах)
    - [ ] Количество выполненных запросов с момента старта
- [ ] **Интеграционные тесты:** Проверить, что связка «запись -> WAL -> поиск -> статистика» работает для 10 000 моментов без вылетов

---

## Этап 2: Графы и связи (Базовый уровень)
- [ ] **Схема хранения связей:** Определить на диске структуру, хранящую список смежности для графа между моментами
- [ ] **Создание связей при записи:** При добавлении нового `Moment` вычислять (по времени или по вектору) и материализовать связи с последними N моментами
- [ ] **Обход графа:** Реализовать API для получения всех моментов, связанных с данным, на глубину 1 (или больше, но хотя бы 1 уровень)
- [ ] **Интеграция с поиском:** При выполнении гибридного поиска добавить опцию «включить связанные моменты» (например, если найден момент X, добавить к результату его соседей по графу)

---

## Этап 3: Завершение (Сборка и документация)

- [ ] **Сборка:** Настроить `cargo build` так, чтобы бинарник содержал весь функционал
- [ ] **Базовый CLI:** Минимальный интерфейс для демонстрации: вставка моментов, поиск, вывод статистики (пока без сложных конфигов)
- [ ] **README:** Описать, как запустить, и показать пример поиска (вход → выход) с флагом гибридного поиска и хронологической сортировкой
